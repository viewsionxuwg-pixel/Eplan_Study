# 学习路线图

学习完成以“实机操作或案例成果通过验收”为准，只有笔记不计为掌握。基线来自 [[README#7. 学习路线|Vault 根文件第 7 章]]。

## 初学者推荐主线

先从 [[00_Home/初学者快速入口|初学者快速入口]] 开始。下面五个阶段是能力地图，不要求在第一周全部展开。

| 轮次 | 范围 | 学习目标 | 通过证据 |
|---|---|---|---|
| 第一轮：最小闭环 | 项目、页、符号、功能、DT、连接、检查、少量部件、报表、备份 | 看懂 EPLAN 的“对象—数据—检查—输出”关系 | 电机启停项目、检查记录、部件表、PDF、`.zw1` |
| 第二轮：工程对象 | PLC、端子、插头、电缆、典型自动化设备 | 文档能够支持控制柜接线与 I/O 核对 | 小型 PLC 控制柜主要原理图和对象报表 |
| 第三轮：复用与标准化 | 宏、占位符、部件主数据、报表配置、模板 | 同类回路可复用，输出可重复生成 | 可切换型号的设备宏和经复核的标准候选 |

第一轮未通过前，不把 Pro Panel、SQL 部件库、API、脚本、大型旧项目转换或公司模板定制加入当前任务。

## 阶段一：基础建模

- [ ] 理解 EPLAN 的数据化设计思想
- [ ] 创建项目并理解项目类型
- [ ] 建立结构标识和页结构
- [ ] 插入符号并理解功能定义
- [ ] 设置设备标识符
- [ ] 理解自动连接、连接定义点和电位
- [ ] 熟悉页、设备、连接和部件导航器

验收输出：一个可检查的接触器控制电机启停小项目。

导航：[[20_基础知识/EPLAN基础设计_MOC|EPLAN 基础设计 MOC]] · [[60_案例工程/案例工程_MOC|案例工程 MOC]]

## 阶段二：典型自动化对象

- [ ] 断路器、接触器、继电器和电源
- [ ] PLC 盒、PLC 连接点和地址
- [ ] 传感器、阀、指示灯和按钮
- [ ] 伺服驱动器和电机
- [ ] 端子排及端子附件
- [ ] 插头和插针
- [ ] 多芯电缆、屏蔽和备用芯
- [ ] 急停、STO 和安全相关信号的文档表达

验收输出：一套小型 PLC 控制柜的主要原理图。安全相关内容必须注明依据、适用边界和待专业复核项。

导航：[[30_工程对象/PLC与IO_MOC|PLC 与 I/O MOC]] · [[30_工程对象/端子插头电缆_MOC|端子、插头与电缆 MOC]]

## 阶段三：部件、宏与复用

- [ ] 理解部件数据结构和部件分配
- [ ] 从 Data Portal 取得并检查部件
- [ ] 自建部件及技术数据
- [ ] 创建窗口宏、符号宏和页宏
- [ ] 创建宏项目
- [ ] 使用占位符对象和值集
- [ ] 建立典型回路库

验收输出：同一电机或伺服控制单元可通过宏复用并切换型号。

导航：[[40_部件与宏/部件与宏_MOC|部件与宏 MOC]]

## 阶段四：报表与制造输出

- [ ] 封面、目录和页总览
- [ ] 部件表和设备清单
- [ ] 端子图表
- [ ] 电缆图表
- [ ] PLC 图表
- [ ] 连接列表
- [ ] 线号、端子号和铭牌数据
- [ ] PDF 及其他格式导出

验收输出：由原理图自动生成一套可用于采购、装配和接线的资料。

导航：[[50_报表与制造/报表与制造_MOC|报表与制造 MOC]]

## 阶段五：公司标准化

- [ ] 定义公司项目结构
- [ ] 定义图框、封面和表单
- [ ] 定义设备编号和页编号规则
- [ ] 定义部件与宏命名规则
- [ ] 建立推荐器件库
- [ ] 建立项目检查方案和设计评审清单
- [ ] 建立归档、版本和交付规范

验收输出：新项目能够从经人工确认的公司基本项目或项目模板启动。

导航：[[70_公司设计标准/公司标准化_MOC|公司标准化 MOC]]
